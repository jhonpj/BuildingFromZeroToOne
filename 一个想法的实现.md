# 核心目标、价值与痛点

这款应用最想解决用户的什么核心痛点？比如 “帮助租房者快速匹配真实房源并规避中介套路”，而非泛泛的 “做一个租房 APP”。

# 目标用户

具体是哪类人群？（例：20-30 岁一线城市职场新人 / 中小餐饮店主 / 学生家长等），是否有明确的用户画像特征？比如 C 端普通用户、B 端企业客户、特定行业从业者等

# 核心功能

您认为支撑 “解决核心痛点” 的 3-5 个核心功能是什么？（例：房源真实核验、直连房东聊天、租房合同模板生成）。


# 竞品认知
目前市场上是否有类似产品？它们的不足或未覆盖的需求是什么？（这是差异化关键）。

# 发布形式

比如 Web 网站、微信小程序、移动端 App，或多端适配


# MVP预期

您希望最小可行产品（MVP）上线时，至少能实现什么效果？能让用户完成什么核心操作？


# 约束条件
是否有开发周期（例：3 个月内上线 MVP）、团队技术栈（例：前端只会 Vue）、预算（例：云服务器月预算 500 元内）等限制？
