## [bun](https://www.bunjs.cn/)

>[!tip]
un 是用于运行 JavaScript 和 TypeScript 应用程序的集成工具包。它以单一可执行文件的形式发布，文件名为 `bun`。
Bun 的核心是 _Bun 运行时_，这是一种快速 JavaScript 运行时，可直接替换 Node.js。Bun 采用 Zig 语言编写，底层采用 JavaScriptCore 引擎，大大减少了启动时间和内存使用量。




## [Docker和Dockerfile](https://labex.io/zh/tutorials/docker-a-beginner-s-guide-to-creating-and-using-dockerfiles-392952)

>[!tip]
>Dockerfile 是一个基于文本的脚本，其中包含一组用于构建 Docker 镜像的指令。它指定了基础镜像、要执行的步骤以及容器的配置设置。通过使用 Dockerfile，你可以自动化创建和管理 Docker 镜像的过程，从而更轻松地构建、分发和部署你的应用程序
>[菜鸟教程](https://www.runoob.com/docker/docker-dockerfile.html)



### [docker-compose](https://www.runoob.com/docker/docker-compose.html)

>[!tip]
>compose、machine 和 swarm 是docker 原生提供的三大编排工具。简称docker三剑客。
>Docker Compose能够在 Docker 节点上，以单引擎模式(Single-Engine Mode)进行多容器应用的部 署和管理。多数的现代应用通过多个更小的微服务互相协同来组成一个完整可用的应用。
>部署和管理繁多的服务是困难的。而这正是 Docker Compose 要解决的问题。Docker Compose 并不 是通过脚本和各种冗长的 docker 命令来将应用组件组织起来，而是通过一个声明式的配置文件描述整 个应用，从而使用一条命令完成部署。应用部署成功后，还可以通过一系列简单的命令实现对其完整声 明周期的管理。甚至，配置文件还可以置于版本控制系统中进行存储和管理。
>[全网最详细的Docker-Compose详细教程](https://juejin.cn/post/7042663735156015140)
>[使用Docker Compose创建多容器应用](https://learn.microsoft.com/zh-cn/visualstudio/containers/tutorial-multicontainer?view=visualstudio)

## [MCP](https://mcpcn.com/docs/tutorials/)

>[!tip]
>MCP（模型上下文协议）是AI与外部世界的统一通信标准，核心价值在于重构AI应用架构。它解决传统"NxM集成困境"——当N个AI应用需连接M个外部服务时，MCP通过标准化协议实现即插即用：用户输入自然语言指令→MCP客户端解析意图→MCP服务器调用工具（如发邮件/查日历）→返回结果。其三大优势是可插拔性（像拼积木添加功能）、可发现性（自动识别可用工具）、可组合性（灵活调度服务）。有了MCP，用户只需清晰表达任务目标，AI便能自主完成工具调度，标志着AI从"生成内容"迈向"调度执行"的新阶段
